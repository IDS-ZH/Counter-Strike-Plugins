# Шпаргалка проекта ZH-Sys (CS:Source, Оллима, ROCm)

Эта шпаргалка автоматически загружается агентом при входе в этот воркспейс. Она содержит выжимку критических правил и набитых шишек.

## 1. Ollama и AMD ROCm (6900 XT)

### Проблема: `unexpected EOF (status code: 500)`
Если Ollama падает с такой ошибкой (часто сопровождается `ROCm error: an illegal memory access was encountered` в логах), это означает, что:
1. Вы используете `format='json'` (JSON Grammar Parser), который очень нестабилен в `llama.cpp` на картах AMD при генерации больших объемов текста или при огромных контекстах.
2. Модель попыталась выйти за пределы аппаратно зашитого `context_length`. (Например, `qwen2.5-coder` залочен на 32k, даже если вы запрашиваете 128k).

### Решение: Проверка честного контекста
Никогда не угадывайте контекст. Всегда проверяйте его через API:
```python
import requests
r = requests.post('http://localhost:11434/api/show', json={'name': 'gemma4:e4b-it-q8_0'})
print(r.json()['model_info'].get('llama.context_length')) # -> 131072 (128k)
```

## 2. Разработка Плагинов (Standalone и ZH-sys)

### Права Доступа (Критическое правило)
- **Гости (Обычные игроки):** Никаких полномочий для управления сервером или другими. Доступны только локальные косметические функции плагинов (например, включение/выключение `Show Damage` или `HUD` исключительно **для себя**).
- **VIP / Friends:** Могут настраивать только определенные параметры плагинов, если они явно прописаны в конфиге плагина. Иначе — прав нет.
- **Full Admins (ROOT):** По умолчанию имеют доступ к настройке всех параметров через чат-команды. 
*Важно:* Настраивать контроль доступа и ограничивать команды нужно на этапе проектирования плагина! Не оставлять уязвимостей (дыр) для "троллей".

## 4. Алгоритм Auto-Compact для больших кодовых баз
Если нужно проанализировать или портировать огромную кодовую базу (как Hammer Editor) с помощью локальной нейронки, используйте подход **OpenCode**:
1. Разбейте проект на чанки (пофайлово или по 25k символов).
2. Заставьте модель выдавать жесткий JSON: `{"previous_knowledge": "...", "new_findings": "...", "final_merged": "..."}`.
3. Сделайте программный (Python) предохранитель: если ответ пустой или содержит ошибку `unexpected EOF`, скрипт не должен падать, он должен откатиться к прошлому `final_merged` (Rolling Summary).
4. Если оперативной памяти много (как у нас, 60+ ГБ), можно держать несколько моделей (Planner, Coder, Vision) в памяти одновременно и вызывать их как мультиагентный конвейер.
