# Development Changelog - SoundManifest плагин

## 2025-11-24

- Localization: chat and HUD messages now format per-client language (`%T`), fixing English-only output in feed/HUD.
- Sound menu title and toggle item text are translated per player instead of showing raw keys.

## 7 ноября 2025

### Основные изменения от Qwen

1. **Создана система отдельных HUD-сообщений**
   - Введены отдельные ключи для HUD сообщений без цветовых тегов (например, `FirstBlood_HUD`, `Headshot_HUD`)
   - Создана функция `GetHudMessageKey()` для преобразования обычных ключей сообщений в HUD-специфичные
   - Изменена функция `ShowHudMessageToAudience()` для использования отдельных ключей перевода
   - Решена проблема "монохромности" HUD-сообщений, связанная с ограничениями библиотеки `easy_hudmessage`

2. **Исправлены предупреждения компилятора**
   - Устранено предупреждение о несоответствии тегов (tag mismatch) между `SoundEventType` и `int`
   - Плагин теперь компилируется без каких-либо предупреждений

3. **Улучшена структура конфигурации**
   - Файл перевода `SM_SoundManifest.phrases.txt` обновлен с отдельными ключами для чата и HUD
   - Ключи для чата содержат цветовые теги (например, `FirstBlood_Message`)
   - Ключи для HUD не содержат цветовые теги (например, `FirstBlood_HUD`)

### Технические детали

- **Исходный файл:** `SoundManifest\QWEN\cstrike\addons\sourcemod\scripting\SoundManifest-V3.2.sp`
- **Скомпилированный файл:** `SoundManifest\QWEN\cstrike\addons\sourcemod\plugins\SoundManifest-V3.2.smx`
- **Ответвление:** От Gemini версии 3.2
- **Статус:** Теперь является основной версией для разработки

### Решенные проблемы

1. **Проблема с цветовыми тегами в HUD**
   - **Проблема:** HUD-сообщения не поддерживали цветовые теги, что приводило к их отображению как обычный текст
   - **Решение:** Создана отдельная система ключей для HUD-сообщений без цветовых тегов

2. **Предупреждения компилятора**
   - **Проблема:** Несоответствие тегов между `SoundEventType` и `int` в проверках
   - **Решение:** Использование `view_as<int>()` для корректного сравнения типов

### Поддерживаемый функционал

- Система селективных уведомлений (банальные убийства не объявляются)
- Поддержка различных аудиторий для разных звуков одного события
- Отслеживание серий убийств и командных убийств
- Система приветственных звуков с поддержкой групп администраторов
- Настройка через CVAR и конфигурационные файлы

## 8 ноября 2025

### Исправления для SoundManifest плагина (относительно обработки плейсхолдеров)

1. **Проблема с неправильным именем конфигурационного файла**
   - **Проблема:** SoundManifest-QWEN использовал неправильное имя файла конфигурации `SoundManifest_Sounds.ini` вместо `SoundManifestConfiguration.cfg`
   - **Решение:** Исправлено имя конфигурационного файла в плагине на правильное

2. **Ошибка в логике обработки плейсхолдеров `{1:N} {2:N}`**
   - **Проблема:** В обоих версиях (QWEN и Gemini) плагин пытался использовать значение из конфигурации как ключ перевода, но если в конфиге было не имя ключа, а готовый текст с плейсхолдерами, система не могла заменить `{1:N} {2:N}` на имена игроков
   - **Решение:** Убедились, что в конфигурационном файле указаны названия ключей перевода, а не готовый текст с плейсхолдерами

3. **Заблуждение Gemini**
   - **Факт:** Gemini думал, что его правки в коде форматирования сообщений "гарантированно" решат проблему с плейсхолдерами
   - **Реальность:** Проблема была в неправильном файле конфигурации и использовании неправильных данных, а не в форматировании сообщений
   - **Результат:** Его изменения частично исправляли визуализацию, но не решали корневую проблему

4. **Интеграция улучшений DeathInformer и SoundManifest**
   - **Преимущество:** Оба плагина теперь используют одинаковую систему обработки данных и могут делиться кодом
   - **Перспектива:** Возможность создания общей библиотеки для обработки событий смерти и повреждений
