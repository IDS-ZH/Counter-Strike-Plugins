# SoundManifest-V3.2 - Изменения в архитектуре

## Основные изменения

### 1. Новая логика воспроизведения событий

#### Ранее (до версии 3.2):
- Все события (включая убийства из стандартного оружия) пытались воспроизводиться
- Отсутствовала четкая дифференциация между "банальными" и "особенными" событиями
- Воспроизводились звуки и сообщения даже для простых убийств из базового оружия

#### Текущая архитектура (с версии 3.2):
- "Банальные" события убийства из стандартного оружия (RifleKill, SMGKill, ShotgunKill, SniperKill, PistolKill, MachineGunKill) НЕ воспроизводят звуки и НЕ показывают сообщения
- Эти события по-прежнему отслеживаются в коде для определения типа оружия, но просто не выводятся игрокам
- "Особенные" события (FirstBlood, Headshot, KnifeKill, GrenadeKill, SpecialKill, серии убийств и т.д.) продолжают работать в полном объеме

### 2. Определение базовых убийств

Функция `IsBasicWeaponKill(SoundEventType eventType)` определяет, является ли событие "банальным":
```sourcepawn
bool IsBasicWeaponKill(SoundEventType eventType)
{
    switch (eventType) {
        case EVENT_RIFLEKILL, 
             EVENT_SMGKILL, 
             EVENT_SHOTGUNKILL, 
             EVENT_SNIPERKILL, 
             EVENT_PISTOLKILL, 
             EVENT_MACHINEGUNKILL:
            return true;
        default:
            return false;
    }
}
```

### 3. Обновление файла конфигурации

Файл `SoundManifestConfiguration.ini` теперь содержит только настраиваемые события, которые воспроизводятся. События базовых убийств из стандартного оружия в него не включаются, т.к. они не требуют воспроизведения.

### 4. Файл переводов

Файл `SM_SoundManifest.phrases.txt` также не содержит фраз для "банальных" событий убийства из стандартного оружия, что соответствует новой логике плагина.

## Преимущества новой архитектуры

1. **Уменьшение спама** - больше нет звуков и сообщений для каждого обычного убийства
2. **Улучшенная производительность** - плагин не обрабатывает ненужные события
3. **Лучшая дифференциация** - акцент на действительно значимые игровые события
4. **Совместимость** - все специальные события, серии убийств, и другие значимые события работают как и раньше

## CVAR плагина

Остались те же CVAR, что и в предыдущих версиях, но теперь с более понятной логикой применения:
- `sm_soundmanifest_enabled` - включить/выключить плагин
- `sm_soundmanifest_join_delay` - задержка приветственного звука
- `sm_soundmanifest_join_audience` - кто слышит приветствие
- `sm_soundmanifest_join_interval` - минимальный интервал между приветствиями
- `sm_soundmanifest_default_setting` - включены ли звуки по умолчанию
- `sm_soundmanifest_justicekill_threshold` - порог убийств товарищей для "справедливого" убийства
- `sm_soundmanifest_combokill_time` - время для комбо-убийств
- `sm_soundmanifest_quadkill_threshold` - порог для QuadKill
- `sm_soundmanifest_epicstreak_threshold` - порог для "эпических" серий
- `sm_soundmanifest_cooldown_interval` - интервал между одинаковыми звуками
- `sm_soundmanifest_max_file_size` - максимальный размер файла
- `sm_soundmanifest_max_sounds_per_event` - максимальное количество звуков на событие
