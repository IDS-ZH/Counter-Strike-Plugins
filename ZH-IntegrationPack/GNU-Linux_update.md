# Интеграция SRCDS в ZH-sys на GNU/Linux

## Стратегия управления сервером
Принято решение **не** использовать готовые "коробочные" решения (панели и лаунчеры) как основу, так как они ограничивают кастомную логику и усложняют тесную интеграцию с ZH-sys. Мы загружаем "ванильный" (чистый) SRCDS от Valve и строим свою архитектуру поверх него. 

Готовые проекты будут использоваться исключительно как **референсы** для изучения лучших практик (например, как они парсят логи, как обрабатывают краши, как организуют RCON-взаимодействие).

### Анализ существующих решений (референсы)

#### 1. Pterodactyl Panel
- **Что это:** Популярная веб-панель управления игровыми серверами на базе Docker (Wings daemon).
- **Чем полезно для ZH-sys (перенятие опыта):**
  - Принципы изоляции ресурсов (ограничение RAM/CPU на инстанс).
  - Красивая реализация Web-консоли (чтение stdout через websocket).
  - Подход к конфигурации через переменные окружения и "яйца" (eggs/startup scripts).
- **Ограничения и неудобства (почему мы не используем её напрямую):**
  - **Докер-зависимость:** Сервер изолирован в контейнере. Изменить параметры старта "на лету" из стороннего PHP/Python-скрипта ZH-sys проблематично (нужно дергать REST API панели, а не просто перезапускать bash-скрипт).
  - Усложненный файловый доступ: копирование плагинов `.smx` напрямую в папки требует работы с volume'ами докера или встроенным SFTP. Это ломает быстрый цикл разработки и отладки (Development -> Compile -> Load).

#### 2. LinuxGSM (Linux Game Server Managers)
- **Что это:** Набор мощных bash-скриптов для запуска, бэкапов и мониторинга серверов (включая `cssserver`).
- **Чем полезно для ZH-sys (перенятие опыта):**
  - Использование `tmux` для работы сервера в фоне и организации консольного ввода/вывода.
  - Логика авто-апдейтов (через `steamcmd`).
  - Сбор метрик (uptime, потребление ресурсов) и интеграция вебхуков для алертов о падениях.
  - Обработка зависаний (watchdog скрипты).
- **Ограничения и неудобства (почему мы не используем её напрямую):**
  - Монолитность и перегруженность функционалом, который может конфликтовать с экосистемой ZH-sys.
  - Затруднительное динамическое управление командной строкой запуска (режимы "для друзей", голосования за моды требуют постоянного перезаписывания конфигурационных файлов LinuxGSM).

## План действий (Текущий этап)
1. Загрузка чистого `srcds` (AppID 232330) через `steamcmd` в `/mnt/1tb_storage/SRCDS/CS_Source/`.
2. Копирование скомпилированных плагинов из `/Standalone/` (включая настроенные конфиги и переводы).
3. Прописывание `STEAMID` разработчика в `admins_simple.ini` для получения полного доступа.
4. Разработка легковесного скрипта-стартера для интеграции с ZH-sys (с поддержкой передачи параметров запуска извне).
